# 5.4　4つの壁 × 2つのアプローチの整理

これまでの整理を踏まえ、本節では、4つの壁それぞれについて、最初のアプローチおよび展開のためのアプローチにおいてどのような観点で向き合うべきかを整理する。
本研究で定義するアプローチは、第4章で整理した課題構造と、変革プロセスにおける壁の関係性を踏まえて整理したものである。

## 各壁に対する2つのアプローチ

| 壁の名称 | 最初のアプローチ | 展開のためのアプローチ |
| :--- | :--- | :--- |
| **自分の壁** | 個人としてAIを業務に取り入れ、自分が納得できるだけの価値と手応えを展開できるように整える。 | 自分の経験や得られた価値・不安を言語化し、推進者として周囲と共有できる形に整理することで、範囲を広げることへの懸念を払拭し、適用範囲を段階的に広げていく。 |
| **他者の壁** | | |
| **資源の壁** | | |
| **環境の壁** | 実際にAIを業務の中に取り入れ、組織として誰が判断するかのルールと安心して使える仕組み、仕事として回し続ける流れを整える。 | 小規模組織で整えたやり方を整理し、誰がやっても同じように判断・運用できる形にして、組織の仕事として定着させる。 |

## 5.4.1　自分の壁

### （1）自分の壁を分解して捉える視点
自分の壁は、「やる気の有無」や「スキルの有無」といった単一の要素ではなく、いくつかの要素が組み合わせ合って現れるものであると捉える。本研究では、自分の壁を次の4つの視点にわけて整理する。

*   **目的・解像度**：何のためにAIで業務をリデザインしたいのか、自分なりのゴールや価値の基準が見えているか
*   **スキル・知識**：AIツールの基本的な使い方や、業務を再設計するときの考え方を理解できているか
*   **感情・心理**：失敗への不安、周囲の目への心配や展開してもよいと自分で思えるだけの納得感があるか
*   **習慣・時間**：日々の業務の中で、AIを試し振り返る時間があるか。継続できるか。

### （2）最初のアプローチ
「これは組織に展開してもよい」と自分で納得できるだけの価値を見出すことを重視する。
小規模な範囲で、自分の業務の一部を題材に「AIを前提としたやり方」を試し、自分自身がその価値とリスクを腹落ちさせることをねらいとする。

**【タスク】**
1. 自分にとっての「広げる価値がある状態」の基準を決める
2. 一つの業務で、AIを組み込んだ「個人としての成功パターン」をつくる
3. 結果とプロセスを簡単に記録し、「何が良かったか／どこに不安が残るか」を言語化する
4. 少人数のメンバーに共有し、フィードバックをもらう

### （3）展開のためのアプローチ
自分の中で見出した価値を、グループや組織に広げていく段階を想定する。
ここでの焦点は、推進者として、「範囲を広げることへの懸念」をどう扱うか、広げることで得られる価値を、周囲と共有できる形でどう見出すかにある。
個人レベルで得られた学びや成功パターンを、誰が見ても分かる「説明」として整理し直すことで、自分一人の思い込みではなく、「組織として検討に値するテーマ」として扱える状態を目指す。

**【タスク】**
1. 個人で得た価値や学びを「なぜ／何を／どうやったか」の形で整理する
2. 範囲を広げるときに想定される懸念・リスクを書き出す
3. その不安・リスクに対する「とりあえずの対策」と「線引き」を決める
4. 上長と対話し、「広げることのメリット」と「リスクと対策」をセットで共有し、フィードバックをもらう

## 5.4.4　環境の壁

### （1）環境の壁を分解して捉える視点
環境の壁は、制度やシステムといった単一の要素ではなく、複数の仕組みが噛み合って初めて機能する領域である。本研究では、環境の壁を次の4つの観点に分けて整理する。

*   **ルール**：誰が責任を持つのか、どこで人が判断するのか、何をしてはいけないのか
*   **基盤**：安心して利用できる環境があるか、利用状況を確認できる仕組みがあるか
*   **運用**：日常的に誰が回すのか、トラブルや改善にどう対応するのか
*   **制度**：評価や役割、組織方針と接続され、個人依存になっていないか

### （2）最初のアプローチ
小規模組織において、完璧な環境を目指すのではなく、業務として回せる最低限のルール・基盤・運用を整え、Lv3が実務上成立する状態をつくる。

**【タスク】**
1. 誰が責任を持つかを決める
2. 人が必ず確認するポイントを決める
3. 安心して試せる場を用意する
4. うまくいったら、次にどうするかを決めておく

### （3）展開のためのアプローチ
小規模組織で成立したやり方を整理・一般化し、関与する人や組織が変わっても同様に運用できる形へと整え、制度との接続を通じて定着を図る。

**【タスク】**
1. ルールを整理する
2. 仕組みを整える
3. 回し続ける流れをつくる
4. 制度につなげる
