# 資源の壁を超えるためのアプローチ

## 1. 資源の壁とは

資源の壁とは、やりたい意思や合意はあるにもかかわらず、時間・人・予算・データといった「実行余力」が現実に出せないために、AI共創の取り組みが前に進まない状態を指します。

資源の壁にぶつかると、取り組みは次のように止まります。
*   忙しさに埋もれ、検証が進まない
*   兼務のままになり、担当者が疲弊する
*   データ準備が後回しになり、精度が上がらない
*   予算の話が出た途端に止まる

> [!IMPORTANT]
> 「やる気」や「価値理解」の問題ではなく、**“本当に出せる余力があるか”**が問われている状態です。

---

## 2. 資源を「4つの観点」で整理する

資源の壁は、次の4つの観点で整理すると見えやすくなります。

| 観点 | 説明 |
| :--- | :--- |
| **① 時間** | 改善や検証に使える時間が確保されているか。日常業務に押し戻されていないか。 |
| **② 人** | 誰が主に担うのかが決まっているか。兼務に無理が集中していないか。 |
| **③ 予算** | PoCや検証に使える予算があるか。毎回ゼロから説明・稟議が必要になっていないか。 |
| **④ データ** | すでに使えるデータがあるか。整備が後回しになり、精度低下の原因になっていないか。 |

このどれかが欠けると、AI活用は「やりたいけれど続かない」状態になります。

---

## 3. 資源の壁に対する「最初のアプローチ」

限られた時間・人・予算・データの中で実際に動かし、どこで無理が出るのか、何が足りずに止まるのかを具体的に見える形にする。

まずは、この取り組みに対して、組織がどこまで余力を出せるのかを実際の業務の中で確かめます。最初からうまく回す必要はありません。むしろ、どこで無理が出るかを見ることが目的です。

### 【ねらい・ゴール】
資源を増やすことでも、体制を整えることでもありません。
「小さくても実際に動かしてみる」ことがゴールです。

*   どこで時間が足りなくなったか
*   誰の負担が一番大きかったか
*   どのデータが足りなかったか
*   予算が必要になるのはどこからか

を、具体的に掴みます。

> [!TIP]
> 「できた／できなかった」ではなく「なぜ止まったか」を言語化するフェーズです。

### 【やること】
1.  **資源が少なくても動かせる領域を選ぶ**
    *   データがすでにある、影響範囲が限定されている、効果が測りやすいテーマを選びます。
    *   👉 無理に広げず、止まり方が見えやすいテーマを選びます。
2.  **効果と負担を軽く見える化する**
    *   何分短縮できたか、どの作業が減ったか、逆にどこに手間が残ったか。
    *   👉 大きなKPIではなく、「次の判断材料」を残します。
3.  **「止まった理由」をそのまま残す**
    *   忙しくて後回しになった、兼務では回らなかった、データ準備に工数がかかった、予算で止まったなど。
    *   👉 これらは失敗ではなく、資源の壁を越えるための材料です。

### ★まとめ
最初のアプローチは、組織がどこまで余力を出せるのかを**“さらけ出すフェーズ”**。ここでは、まだ整えません。

---

## 4. 資源の壁に対する「深化のためのアプローチ」

見えてきた制約や偏りをもとに、どこに時間を使うか、誰の工数とするか、何を投資として認めるかを組織として決める。

最初のアプローチで見えてきたのは、「頑張ればできるが、続けられないポイント」です。深化では、そこに向き合い、出す／出さない資源をはっきり決める段階に進みます。

### 【ねらい・ゴール】
資源を無制限に増やすことではありません。
「資源配分の優先順位を、組織として決めること」がゴールです。

*   本当に時間を割くのか
*   誰の工数として認めるのか
*   どこまでを投資として受け入れるのか
*   どのデータ整備を優先するのか

### 【やること】
1.  **時間を「善意」ではなく前提にする**
    *   改善や運用に使う時間を明示する。
    *   👉 忙しさで吸収されないようにします。
2.  **人の役割を最低限固定する**
    *   現場のオーナー、支援する役割（IT／推進など）。
    *   👉 兼務でも、「誰が何を担うか」を曖昧にしません。
3.  **予算を“小さな枠”として切り出す**
    *   PoC・検証用の小口枠を用意する。
    *   👉 毎回ゼロから止まらないようにします。
4.  **データ整備の優先順位を決める**
    *   全部やらない。勝てる領域から整える。
    *   👉 データ不足で「使えない認定」されるのを防ぎます。

### ★まとめ
深化のためのアプローチは、**資源をどう配るかを決めるフェーズ**。ここで初めて、「本気で進むかどうか」が明確になります。

---

## 5. 環境の壁との違い（補足）

| 壁 | 焦点 | 役割 |
| :--- | :--- | :--- |
| **資源の壁** | 出せる余力はあるか？ | 覚悟と優先順位を明らかにする段階 |
| **環境の壁** | 出した資源が回るか？ | 仕組みとして整える段階 |

この順番だからこそ、環境の壁に進む意味が生まれます。
